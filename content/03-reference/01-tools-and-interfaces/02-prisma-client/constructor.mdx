## Overview

This page describes the Prisma Client constructor parameters.

## `datasources`

Use the `datasources` constructor parameter to programatically overwrite

1. Define a datasources in the `prisma.schema` file. In the following example, there is a single datasource named `my_database`:

   ```prisma
   datasource my_database {
   provider = "sqlite"
   url      = env("DATABASE_URL")
   }
   ```

   The `DATABASE_URL` environment variable defines the following value:

   ```
   DATABASE_URL="file:./dev.db"
   ```

   > **Note**: You must re-generate the Prisma Client each time you add or rename a datasource.

2. The following example demonstrates how to overwrite the connection string of the `my_database` datasource:

   import { PrismaClient } from '@prisma/client'

   ```ts
   const prisma = new PrismaClient({
     datasources: {
       my_database: 'file:./dev_qa.db',
     },
   })
   ```

> **Note**: If you do not specify a datasource in the constructor, Prisma Client uses the first datasource in the `prisma.schema` file - in this case, `db`.

## `log`

See logging.

```ts
const prisma = new PrismaClient({
  log: [{ emit: 'event', level: 'query' }, 'info', 'warn'],
})
```

## `errorFormat`

```ts
const prisma = new PrismaClient({
  errorFormat: 'minimal', // Defaults to 'pretty'
})
```
